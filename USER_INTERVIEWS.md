\# USER INTERVIEWS



\# Interview 1 — Engineering Student Using AI Coding Tools



\## Profile



\* Final-year engineering student

\* Uses ChatGPT, Cursor, and GitHub Copilot

\* Works on freelance and academic projects



\## Key Insights



The user subscribes to multiple AI tools simultaneously because each tool feels useful in different situations.



However:



\* they do not track total monthly AI spend

\* they underestimate combined subscription costs

\* they often forget inactive subscriptions



The user said:



> “I pay for tools individually without thinking about the total.”



\## Opportunity Identified



Users need:



\* consolidated visibility

\* spend awareness

\* simpler comparison across vendors

\# Interview 2 — Startup Founder



\## Profile



\* Small SaaS startup founder

\* Team size: 8

\* Heavy AI tooling usage across developers



\## Key Insights



The founder reported:



\* developers independently choosing tools

\* overlapping subscriptions

\* unclear ROI across vendors



The biggest frustration was:



> “We adopted tools quickly without a centralized strategy.”



\## Opportunity Identified



The audit tool can act as:



\* spend visibility layer

\---



\# Common Patterns Across Interviews



\## 1. AI spend grows invisibly



Most users underestimate cumulative spend across tools.



\---



\## 2. Tool overlap is extremely common



Users subscribe to:



\* multiple coding assistants

\* multiple chat models

\* multiple API vendors



without clear optimization.



\---



\## 3. Few users know enterprise pricing



Most users:



\* do not understand enterprise tiers

\* cannot estimate whether they are overpaying

\* do not benchmark against similar teams



\---



\## 4. Users respond strongly to savings visuals



During testing, users reacted much more positively to:



\* savings breakdowns

\* current vs optimized spend charts

\* concrete monthly savings numbers



than to generic AI recommendations.



\---



\# Product Decisions Influenced by Interviews



\## 1. Added custom tool support



Multiple users mentioned niche or emerging AI tools not originally listed.



\---



\## 2. Focused heavily on financial clarity



The UI prioritizes:



\* current spend

\* optimized spend

\* yearly savings



because users care most about immediately understandable ROI.



\---



\## 3. Reduced onboarding friction



Users strongly preferred:



\* no login

\* instant audit generation

\* minimal required fields



This reinforced the decision to avoid authentication for the MVP.



\---



\# Biggest Lesson



The interviews reinforced that:

AI adoption inside startups is happening faster than procurement discipline.



That creates a strong opportunity for tooling that helps teams:



\* understand spend

\* reduce waste

\* benchmark usage

\* optimize vendor choices
